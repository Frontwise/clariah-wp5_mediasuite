Help & Feedback

What is the Media Suite?

The CLARIAH Media Suite is a (re)search environment created as part of the larger CLARIAH project, Media Studies focus. It offers a group of specialized services for media scholars (and other scholars who make intensive use of audio-visual media) to enable them to conduct data-driven research. This environment has been built based on a series of digital humanities tools developed in prior projects, such as AVResearcherXL, Comerda, DIVE, Oral History Today (OHT) and TROVE. It aims to make these tools sustainable.

How can I access the Media Suite?

To access the MediaSuite services (from May, 2017) you must have a valid university account in The Netherlands. You will be prompted to fill-in the details via an Identity Provider discovery service. For security and research purposes, we additionally request you to fill in a small questionnaire before accessing. NOTE: During April 2017 you will only be able to access the Media Suite through OneGini. In order to receive an invitation please follow the instructions on this document.

How does the CLARIAH Media Suite work?

The Media Suite is built based on a co-development approach based on the collaboration between scholars and software engineers. In its current version (See releases) is, and does not intend, to be a complete or finished product. The instruments built and offered in the Media Suite allows scholars to identify collection of interest, search, explore, select, visualize and analyze (annotate) these sources via pre-defined tools (see Recipes) or by building their own tools (see Components), by using their own work or user space. Read more our Knowledge Base.

How has the Media Suite been built?

A generic infrastructure is required to cater for the general needs of every user group. The infrastructure provided by the CLARIAH Media Suite incorporates flexible functionality capable of addressing very specialized research questions. Media scholars expressed their desire to use the collections and tools which were previously "locked" together in the individual prototypes. CLARIAH MS has been designed in a modular way (separating data and functionalities from the original tools) each module (or component) performs a single, well-defined task. Modules can interoperate to construct more sophisticated functionality when they are combined in "recipes".

Where is the Media Suite hosted?

Prototypes are currently hosted on a new infrastructure at the The Netherlands Institute for Sound and Vision (NISV) data centre (see the CLARIAH Lab). All tools and components developed within CLARIAH are open source, the Media Suite code is available in the GitHub repository.
